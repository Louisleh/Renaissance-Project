PRD:
#1 PRD: Product Requirements Document (PRD): Renaissance Skills 1. Product Vision & Core Philosophy "Renaissance Skills" is an assessment and development platform built on the premise that the era of hyper-specialization is ending. As the economic logic of specialization that dominated the past two hundred years fades, AI is effectively "killing the factory". The future belongs to the orchestrators and the synthesizersŃthose who possess cross-domain pattern recognition and the ability to connect disparate ideas. The platform aims to help users map their current skill distribution, identify areas of over-specialization, and build a holistic, well-rounded "Renaissance" profile to thrive in an AI-leveraged world. 2. Target Audience Specialists realizing their narrow expertise is vulnerable to automation. Students and young professionals looking to build resilient careers rather than following the outdated playbook of narrow credentialing. Existing generalists looking to formalize and strengthen their cross-domain synthesis skills. 3. Core User Flow Landing: User arrives at a visually striking, dark-mode landing page featuring a faded-gold Vitruvian Man centerpiece, surrounded by a 360ˇ interactive skill graph. Assessment: User selects one of three assessment methods to map their baseline skills. Profiling: The system generates a visual skill graph and assigns a Renaissance Archetype (e.g., The Polymath, The Strategist, The Builder). Development Path: The user is presented with a customized curriculum to balance their skill graph, focusing on their weakest domains to achieve true well-roundedness. 4. Key Features & Modules Module 1: The Interactive Hero & Landing Visuals: Deep charcoal/black backgrounds with elegant, glowing faded-gold accents and typography. Centerpiece: A central "Vitruvian Man" graphic (placeholder for now) surrounded by an interactive, pulsating radar/spider chart mapping domains like Leadership, Creativity, Tech Proficiency, Data Analysis, Critical Thinking, etc. Module 2: Tri-Modal Assessment Engine The Quick Pulse: A rapid 10-question multiple-choice assessment for immediate onboarding. The Deep Dive: A comprehensive, scenario-based evaluation for high-accuracy profiling. The LLM Mirror: A copy-paste prompt generator. The user takes a custom prompt generated by the site, feeds it into their own private local LLM (to maintain privacy of their personal data/chat history), and pastes the LLM's summary back into the platform to generate their skill graph. Module 3: Archetype Categorization Dynamic assignment into categories (as seen in the mockup) based on the assessment data, visually highlighting their dominant traits alongside their weak spots. Module 4: The Synthesis Curriculum (Development) Manual Selection: Users can browse modules in their weak areas. AI-Optimized Path: An automated roadmap that feeds the user micro-courses and reading materials designed specifically to pull their lowest scores up and enhance cross-domain pattern matching. Module 5: Future Monetization / Expansion (Phase 2) Affiliate links to foundational books (e.g., system thinking, history, broad sciences). Integration of 1-on-1 coaching sessions. 

Original prompt:
#2 Prompt:
You are an expert frontend developer and UI/UX designer. Your task is to build a high-fidelity, interactive, single-page prototype for a platform called "Renaissance Skills".

Write the entire application in a single file containing vanilla HTML, CSS, and JavaScript. DO NOT use any external frameworks or libraries (no React, no Tailwind, no Chart.js). All styling and interactivity must be custom-written to ensure maximum performance and lightweight execution.

CORE THEME & DESIGN SYSTEM:
- Theme: "The Rebirth of the Renaissance Man".
- Colors: Deep charcoal/almost-black background (#0d0d0d), with elegant, faded gold accents (#d4af37, #b5952f), and soft off-white text for readability (#e6e6e6).
- Typography: Use elegant serif fonts for headings (like 'Playfair Display' or 'Georgia') and clean sans-serif for body text.
- Vibe: Premium, academic, mysterious, and highly intelligent.

REQUIRED SECTIONS & FUNCTIONALITY:

1. HERO & NAVIGATION:
- A sleek top navigation bar (Home, Assessment, Development, About).
- A hero section centered around a "Vitruvian Man" concept. Since you cannot generate images, create a beautiful, circular, glowing CSS-based radar/spider chart outline in gold.
- Around this central graph, place text labels for skills: Leadership, Creativity, Strategy, Tech Proficiency, Problem Solving, Critical Thinking, Adaptability, Data Analysis.
- Include a prominent call-to-action button: "Start Your Personalized Assessment".

2. TRI-MODAL ASSESSMENT SECTION:
- Below the hero, create a container with three distinct assessment options (styled as elegant gold-bordered cards).
- Option 1: "The Quick Pulse" (Short version).
- Option 2: "The Deep Dive" (Detailed version).
- Option 3: "The LLM Mirror" (Explain that this generates a prompt for the user's private LLM to summarize their skills based on chat history).

3. ARCHETYPE PROFILING SYSTEM:
- A section displaying 4 distinct profiles: "The Polymath", "The Strategist", "The Builder", and "The Leader".
- Use CSS grid/flexbox to arrange these elegantly. Give each a brief description (e.g., "The Builder: High execution and technical skills").

4. DEVELOPMENT & CURRICULUM SECTION:
- A dashboard area simulating the "next steps" after an assessment.
- Include a toggle or two distinct UI areas:
  a) "Targeted Growth": A grid of courses focusing on weak areas.
  b) "AI-Optimized Synthesis Path": A timeline/roadmap view suggesting a specific order of topics to achieve well-roundedness.
- Include placeholders for future features: "Recommended Reading" and "Book a Coaching Session".

INTERACTIVITY (Vanilla JS):
- Add a smooth scroll effect to the navigation links.
- Add a hover effect to the skill labels in the hero section that makes them glow or scale slightly.
- Implement a simple click event on the "Start Your Personalized Assessment" button that smoothly scrolls the user down to the Tri-Modal Assessment Section.

Ensure the final output is responsive, visually stunning, and immediately runnable in a browser. Use placeholder inline SVGs or CSS shapes if icons are needed. Output only the code block.
